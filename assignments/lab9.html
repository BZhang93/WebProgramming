Lab 9: Running Status Updates (Persistent)

This lab is worth 3 points.

Lab Objectives

  • Practice with data persistence and local storage.

Instructions

Modify your Lab 5, your status_dom/profile.html file, so that messages entered (that is, on pressing the enter / return key) will be displayed dynamically (i.e., no page refresh) in the "Updates" section --with the latest update showing first! More importantly, the messages are stored in local storage so if you leave the page, visit other websites, and return to the page, your messages are preserved and are redisplayed!

All the requirements in Lab 5 are still in effect. All displayed messages must be preceded with a timestamp in the format MM/DD/YYYY HH:MM. A space followed by a hyphen followed by a space must follow the timestamp. Thus, the format of a message to be displayed must be:

MM/DD/YYYY HH:MM - the message

The time must be in 24-hour format! You do not need to display the leading zero for either the date or for the time (example: "9" is acceptable for "09").

Viewing Your Work

Open your profile.html page on any web browser that supports local storage.

Submitting the Lab

Commit and push your lab to your private repo.

Assessment

  1. (1 point) Messages are stored in local storage
  2. (1 point) Messages are redisplayed (from local storage) on return to the page
  3. (1 point) Did not change format and display of messages from Lab 5. That is, messages are displayed in the "Updates" section; latest update message is displayed first, at the top; formatting of message
  4. (-1 point) Outputs exist in JavaScript console
