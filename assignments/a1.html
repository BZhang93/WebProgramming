Part 2: Setting Up Your Tufts CS Web Directory

(Most of this document is from the Tufts CS & ECE Systems User Guide)

Tufts CS provides a cluster for Unix logins (as well as two workstation labs running Red Hat Enterprise Linux in Halligan 116 and 118):

  • linux.cs.tufts.edu

If you have not done so already, activate or reset your Tufts CS Unix account at https://www.eecs.tufts.edu/~accounts/.

With your Tufts CS account, you can create a personal website that will be available at:

http://www.cs.tufts.edu/~user_name/

or

http://www.ee.tufts.edu/~user_name/

In order to setup your website, you need to create a folder called public_html in your "home directory" (which is usually your current location after you have successfully logged on). This folder will appear to be the root of your personal website.

You will set the correct permissions on your "home directory" and the public_html directories. This can be accomplished by executing the following commands (assume that your login / username is lsimpson and your command prompt is >):

  1. Log on to linux.cs.tufts.edu
  2. lsimpson ~/work/project/ > cd ~
  3. lsimpson ~/ > mkdir public_html
  4. lsimpson ~/ > chmod 755 public_html
  5. lsimpson ~/ > chmod 711 ~

The default file for your website must be named index.html. Without this file, visitors to your website will see a listing of the files you have in your public_html directory.

As you create new files and directories, you must verify that they have the proper permissions set. The proper setting for files is 644, and directories is 755.

You can set these by executing:

  1. lsimpson ~/ > chmod 755 <directory_name>
  2. lsimpson ~/ > chmod 644 <file_name>

If one of these steps is not completed, you will get the 403 error (status code).

Logging On to Tufts CS System

Microsoft Windows

1. Download SSH Secure Shell: SSHSecureShellClient-3.2.9.exe. SSH Secure Shell provides support for both shell access and file transfer. This is the last version of the SSH Secure Shell available free for non-commercial and university use. This is no longer available through SSH.com ($ reason).

2. Add a "profile" for Tufts CS (you can name it whatever you want).

3. Edit the profile that you created. Enter the host name and your user name (your Tufts CS Unix login).

4. Connect to the server by selecting the profile. You may be prompted to accept a key --do so. Then enter your password.

5. You are in! Your screen should look like the following:

5A. You can now run a plethora of packages and Unix commands. See below.

6. Follow the instructions above to create your public_html directory.

7. Use the Secure File Transfer client that comes with SSH Secure Shell to upload files to your account. You can drag-and-drop files from your computer to your account on the Tufts CS server.

NOTE: SSH Secure Shell is no longer supporter by SSH. Another alternative: PuTTY.

Mac OS X, Linux

SSH and SCP directly from Terminal via ssh username@hostname. Example: ssh mchow@linux.cs.tufts.edu. To find Terminal, open Hard Drive -> Applications Folder -> Utilities Folder -> Terminal).

Secure File Transfer via FileZilla (Mac OS X, Windows, Linux)

Secure file transfer programs such as sftp and FileZilla should work without trouble on the CS systems. FileZilla is recommended for Mac OS X and Linux for secure file transfer. It is also popular for Windows.
