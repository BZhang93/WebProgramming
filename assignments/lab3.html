Web Programming

Lab 3: OMFGDOGS

This lab is worth 3 points.

Lab Objectives

  • Practice writing HTML.
  • Practice writing CSS.
  • Start to understand the benefits of HTML5.

Overview

OMFGDOGS (http://www.omfgdogs.com/) was one of the best useless websites in 2012. It made a splash at ROFLCon last May at MIT. OMFGDOGS is now one of the great memes along with the Hampster Dance and Nyan Cat. One problem: OMFGDOGS uses Flash. It doesn't have to be.

Instructions

Using HTML5 and CSS, recreated OMFGDOGS without Flash by writing two files: index.html and omfgdogs.css. The animated gif shall cover the entire body of the web browser and the tune shall play on page load.

Two assets will be provided for you:

  1. The animated GIF omfgdogs.gif
  2. The tune omfgdogs.mp3

You must also put a link to the original OMFGDOGS website http://www.omfgdogs.com/ somewhere at the bottom-right of the page to give credit where credit is due.. I urge you to use a bigger text size, different font, text colors other than black or blue.

Important Notes

  1. The tune must loop. That is, it is replayed when it is done playing.

Getting Started

  1. If you haven't done so already, clone a copy of the running-dogs repository that you forked in Lab 1. Assuming that you have created your keys correctly for secure communications between your computer and GitHub, the SSH URI for the repository should be something like git@github.com:YOUR_GITHUB_USERNAME/running-dogs.git. Thus, run: git clone git@github.com:YOUR_GITHUB_USERNAME/running-dogs.git
  2. Inside the running-dogs repo, run git remote add upstream https://github.com/tuftsdev/running-dogs.git to keep track of the original repo that I created.
  3. Inside the running-dogs repo, run git fetch upstream and git merge upstream/master to pull in the changes that I made to the original running-dogs repo.
  4. The animated GIF and the MP3 files are in the assets folder.
  5. Create your files index.html and omfgdogs.css in running-dogs. Do not put the files anywhere else.

Open the index.html page on Chrome, Firefox, or the Safari web browsers.

Should you want to make your work viewable on the web, either push all the files to your special repo on GitHub (see Assignment 1) or upload all files to your public_html directory in your Tufts CS account.
