Web Programming

Lab 4: Dice or No Dice

This lab is worth 3 points.

Lab Objectives

  • Practice writing HTML.
  • Practice writing CSS.
  • Practice with responsive design.

Overview

The following are screen resolutions for popular devices available:

  • iPhone 4: 640×960px
  • iPhone 5: 640×1136px
  • iPad 2: 1024x768px
  • iPad 3+: 1536x2048px
  • Nexus 7: 1280×800px
  • Samsung Galaxy S III: 1280×720px
  • Many devices prior to 2011: 320×480px

Instructions

Construct a single HTML page index.html that:

  1. When loaded on a device of your choosing, the word "Dice" is rendered. A CSS sheet dice.css is used.
  2. When loaded on another browser, the words "No dice" are rendered. A CSS sheet nodice.css is used.

Hints and Notes

  1. There is a CSS attribute-value pair display: none that you can use to hide elements of a page. See http://www.w3schools.com/css/css_display_visibility.asp
  2. You are not allowed to use any JavaScript for this lab. (duh!)

Getting Started

You have started Assignment 1 by now, right?

Assuming that you have created your special repo in GitHub username.github.com, create a folder inside of your special repo and name it responsive. Inside the folder shall be the files index.html, dice.css, and nodice.css

Viewing Your Work

Go to http://username.github.com/responsive on a desktop web browser. Also view the page in the browser of a device with the screen resolution you chose.

Submitting the Lab

Make sure all the files are pushed to your special repo in GitHub

Critical: Please make sure you have a README file that notes the device and its screen resolution you chose to use!

Assessment

  1. (1 point) index.html under folder responsive in your repository username.github.com
  2. (1 point) dice.css
  3. (1 point) nodice.css
  4. (-1 point) No README file
