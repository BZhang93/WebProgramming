Web Programming

Lab 7: Friends of Ming Chow Jobs Listing

This lab is worth 3 points.

Lab Objectives

  • Practice using JavaScript associative arrays.
  • Practice parsing JSON data.

Overview

It's all about who you know --and who they know --and who knows you.

Getting Started

  • Assuming that you have set up a local copy of your private repo on your computer, create a new folder inside of it called jobs.
  • It is expected that you work off of the same folder on your computer that is set up to your private repo. You can verify this by viewing the .git/config file and see what the remote URL is!
  • Copy-and-paste the HTML of this page (or right-click and save) to an HTML file named jobs.html in your jobs folder.

Instructions

Parse the string to JSON and render all the jobs on the page in the <div id="list"> section of the body. For each job, show only the following fields (preferably in order):

  1. Company
  2. Location
  3. Position

For this lab, I will encourage creativity: you are free to display all the jobs on the page however you want! You can display all the jobs in a table; or do something more simple or more elaborate. You are encouraged to use stylesheet(s).

However, you are not allowed to add additional HTML tags inside of the <body>...</body>!

Example

Viewing Your Work

Open your jobs.html page on any web browser.

Debugging

Be sure to look at your JavaScript console (e.g., in Chrome Developer Tools or in Firebug)

Submitting the Lab

Commit and push your files to your private repo. The required files that must be in your private repo under jobs are:

  1. jobs.html
  2. README.txt or README.md

You must also include any additional files such as stylesheets.

Assessment

  1. (1.5 points) Parsed JSON correctly
  2. (1.5 point) Rendered all jobs on the page
  3. (-0.5 point) Outputs exist in JavaScript console
  4. (-1.0 point) You added additional HTML tags inside the body!

Hints and Notes

  1. There is something missing in the markup, necessary to make your script code run on page load!
  2. You may be asking yourself: isn't there a better way to do this? That is, shouldn't the data be pulled from a web server? Yes, of course --that is where AJAX is necessary, and the topic of next class!
  3. Consider this your first test if you want to get a job from me. :-)
