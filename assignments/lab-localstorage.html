(Part 1) Lab Objectives

  • Practice with data persistence and local storage.

(Part 1) Instructions

Modify your Lab 5, your status_updates/profile.html file, so that messages entered (that is, on pressing the enter / return key) will be displayed dynamically (i.e., no page refresh) in the "Updates" section --with the latest update showing first! More importantly, store the messages in local storage so if you leave the page and return to the page, your messages are preserved and are redisplayed!

All the requirements in Lab 5 are still in effect. All displayed messages must be preceded with a timestamp in the format MM/DD/YYYY HH:MM. A space followed by a hyphen followed by a space must follow the timestamp. Thus, the format of a message to be displayed must be:

MM/DD/YYYY HH:MM - the message

The time must be in 24-hour format (i.e., military time)! You do not need to display the leading zero for the month in the date or for the hours in time (example: "9" is acceptable for "09").

(Part 1) Viewing Your Work

Open your status_updates/profile.html page on any web browser that supports local storage.

(Part 2) A Preview of Server-Side Development

When you return from spring break, we will move away from client-side development and focus on server-side development. Server-side development is a lot more involved than what we have been doing so far and requires a number of components and software including Node.js and MongoDB (database). Therefore to be prepared after the break, please:

  1. Install Node.js for your computer: http://nodejs.org/. If you have a Mac, I strongly recommend using Homebrew instead of downloading the latest version of Node.js from the Node.js website --see http://mxcl.github.com/homebrew/. If you are using my virtual machine, Node.js is already installed on it.
  2. Create a Heroku account: http://www.heroku.com/
  3. Read up to "Advanced HTTP features", complete the app in the tutorial (up to "Advanced HTTP features"): https://devcenter.heroku.com/articles/nodejs
  4. In a README.txt or README.md for this lab, put the (Heroku) URL of your Node.js app (e.g., http://sharp-rain-871.herokuapp.com/).

Submitting the Lab (Both Parts)

Commit and push your updates to your private repo under status_updates, same folder as Lab 5. That is, no need to create a new folder for this lab.
