Web Programming

Lab: Fork Me, Fork You

Lab Objective

  1. Practice using Git and GitHub

Instructions

  1. Create a GitHub account if you do not have one already at https://github.com/.
  2. Read https://help.github.com/articles/set-up-git on setting up Git on your computer.
  3. Read https://help.github.com/articles/create-a-repo on creating a Git repository on GitHub, adding files to your repository, committing, and pushing files to GitHub. Please create the hello-world repository under your account!
  4. Read https://help.github.com/articles/fork-a-repo to understand what forking in GitHub is, and how to fork a repository.
  5. Fork the project tuftsdev/ForkMeForkYou in GitHub (at https://github.com/tuftsdev/ForkMeForkYou).
  6. Assuming that you successfully cloned your fork of the tuftsdev/ForkMeForkYou repo into the current directory in terminal, create a README.txt (must be named that) that briefly describes the events and files that were once in this repo. Hint: run git log in the repository. You may also look into using git reset or git checkout. More help at http://stackoverflow.com/questions/4114095/git-revert-to-previous-commit-how. You must push the README.txt to your forked repo.

Assessment

This lab is worth 3 points.

  • (1 point) Successfully created the hello-world repo
  • (1 point) Successfully forked the tuftsdev/ForkMeForkYou repo
  • (1 point) Successfully pushed a README.txt to your fork of the tuftsdev/ForkMeForkYou repo that describes the past events and files in the forked repo.
