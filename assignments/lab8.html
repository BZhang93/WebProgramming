Web Programming

Lab 8: Friends of Ming Chow Jobs Listing Revisited

This lab is worth 3 points.

Lab Objectives

  • Practice using AJAX.
  • Practice handling exceptions.
  • Practice parsing JSON data via API.

Overview

Same idea as Lab 7, but this time, you are to get the jobs data via web-based JSON API (read: remotely) using AJAX.

Instructions

You are to modify your lab 7 work. That is, you must modify jobs.html inside your in your jobs folder.

First, remove the long JSON string from the parse() function.

The JSON API of jobs data is at http://messagehub.herokuapp.com/lab8.json. Important: Cross-Origin Resource Sharing (CORS) is enabled for this API!

Parse the string to JSON and render all the jobs on the page in the <div id="list"> section of the body. For each job, show only the following fields (preferably in order):

  1. Company
  2. Location
  3. Position

You are not allowed to use jQuery for this lab! Also like in lab 7, you are not allowed to add additional HTML tags inside of the <body>...</body>!

Hint

You should reuse most of your code from lab 7 for this lab (i.e., for parsing the response text which is a JSON string).

Viewing Your Work

Open your jobs.html page on any web browser.

Debugging

Be sure to look at your JavaScript console (e.g., in Chrome Developer Tools or in Firebug)

Submitting the Lab

Commit and push your files to your private repo. The required files that must be in your private repo under jobs are:

  1. Updated jobs.html
  2. Updated README.txt or README.md

You must also include any additional files such as stylesheets.

Assessment

  1. (1 point) Got JSON data from web API correctly via AJAX.
  2. (1 point) Rendered all jobs on the page
  3. (1 point) Handled exceptions correctly
  4. (-1.0 point) You added additional HTML tags inside the body!
  5. (-3.0 point) You used jQuery!
